📝 SOP-302: Stock Adjustment Protocols

Standard Operating Procedure for Inventory Adjustments

Purpose

To establish controlled procedures for making inventory adjustments that maintain accuracy while providing proper authorization and audit trails.

Types of Adjustments

Quantity Adjustments

  • Cycle count discrepancies
  • Physical damage or loss
  • Found inventory
  • Unit of measure corrections
  • System transaction errors

Value Adjustments

  • Cost corrections from purchasing
  • Obsolescence write-downs
  • Market value adjustments
  • Currency fluctuation impacts

Location Adjustments

  • Items found in wrong locations
  • Location consolidation moves
  • Warehouse reorganization transfers
  • Staging area corrections

Authorization Matrix

Warehouse Staff Level

  • Up to $100: Self-authorized with documentation
  • Damaged goods up to $250 with photo evidence
  • Location transfers unlimited with scan verification
  • Must document reason code for all adjustments

Supervisor Level

  • Up to $1,000: Review and approve within 24 hours
  • Cycle count variances requiring investigation
  • Batch adjustments for multiple items
  • Customer claim-related adjustments

Manager Level

  • Up to $10,000: Detailed review required
  • Obsolescence and write-off decisions
  • Vendor return authorizations
  • Policy exception approvals

Director Level

  • Over $10,000: Board review for large amounts
  • System-wide inventory revaluations
  • Accounting policy changes
  • External audit adjustments

Adjustment Process

Documentation Requirements

  • Complete adjustment request form in QR Legends system
  • Photograph evidence when applicable
  • Provide detailed reason and business justification
  • Include supporting documentation (reports, emails, etc.)

Investigation Steps

  • Review transaction history for affected items
  • Check for pending transactions or reservations
  • Verify count accuracy through recount if needed
  • Consult with other departments as appropriate

System Processing

  • Enter adjustment with proper reason codes
  • Attach all supporting documentation
  • Process during designated adjustment windows
  • Generate confirmation reports for approvers

Reason Codes

Standard Reason Codes

  • CC - Cycle Count Variance
  • DM - Damaged Inventory
  • FI - Found Inventory
  • LI - Lost/Missing Inventory
  • OB - Obsolete Write-off
  • SE - System Error Correction
  • UM - Unit of Measure Correction
  • VR - Vendor Return

Special Circumstances

  • TH - Theft or Security Incident
  • WD - Weather/Disaster Damage
  • QI - Quality Issue Rejection
  • CR - Customer Return Processing
  • AU - Audit Adjustment

Controls and Monitoring

Monthly Review Process

  • Analyze adjustment trends by reason code
  • Review high-dollar adjustments for patterns
  • Identify process improvement opportunities
  • Update procedures based on common issues

Audit Trail Requirements

  • Maintain complete documentation for 7 years
  • Track all approver signatures and dates
  • Preserve original supporting evidence
  • Provide monthly summary reports to finance

Performance Targets

  • Monthly adjustments under 1% of total inventory value
  • 90% of adjustments processed within approval timeframes
  • Zero unauthorized adjustments
  • 100% documentation compliance for all adjustments
